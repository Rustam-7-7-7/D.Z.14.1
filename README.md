## Использование:

В пакете src создан модуль с названием main.py. В нём реализованы следующие функциональности:

### Классы
Созданы классы 
Product
 и 
Category
.
Для класса 
Product
 описаны необходимые атрибуты.
Для класса 
Category
 определены необходимые атрибуты.

### Атрибуты класса
Для класса 
Category
 добавлены два атрибута класса: количество категорий (
category_count
) и количество товаров (
product_count
).
Атрибуты класса заполняются автоматически при инициализации нового объекта.

### В рамках домашнего задания 14.2 внесены следующие изменения: 

#### Режимы доступа к атрибутам
Атрибут списка товаров класса 
Category
 имеет приватный режим доступа.
Реализован метод 
add_product()
 для добавления продукта в атрибут 
products
.
Метод 
add_product()
 принимает на вход аргументы 
self
 и продукт для добавления.
В приватный атрибут 
products
 продукт добавляется через метод 
append
.
Метод 
add_product()
 не возвращает значения.
Метод 
add_product()
 в классе 
Category
 прибавляет 1 к класс-атрибуту «счетчик продуктов».

#### Геттер
Реализован геттер для приватного атрибута 
products
 с использованием декоратора 
@property
.
Геттер возвращает строку со всеми продуктами в приватном атрибуте 
products
.
Имя метода-геттера — 
products
.
Каждый продукт в геттере сформирован как строка строго по шаблону: 
"Название продукта, X руб. Остаток: X шт.\n"
.
#### Класс-метод
Реализован класс-метод 
new_product()
 с помощью декоратора 
@classmethod
.
Метод 
new_product()
 принимает на вход два аргумента — 
cls
 и продукт для создания в виде словаря — и возвращает экземпляр класса 
Product
 на основе данных словаря.

#### Работа с атрибутом цены
Атрибут «цена» класса 
Product
 имеет приватный режим доступа.
Реализован геттер для атрибута «цена» с использованием декоратора 
@property
. Геттер возвращает значение приватного атрибута цены.
Название метода-геттера — 
price
.
Для приватного атрибута «цена» реализован сеттер в классе 
Product
 с использованием декоратора 
@price.setter
.
Сеттер принимает два аргумента — 
self
 и новое значение цены.
В сеттере реализована проверка на положительное значение новой цены.
Если новое значение цены положительное, то в атрибуте обновляется значение цены.
Если новое значение цены не положительное, то новое значение приватному атрибуту цены не присваивается и выводится сообщение «Цена не должна быть нулевая или отрицательная» в консоль.
Сеттер не возвращает никаких значений.
Название метода-сеттера — 
price
.
